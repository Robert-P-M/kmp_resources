Reports duplicate resource keys within the same Compose Multiplatform XML file.

When defining resources in Compose Multiplatform, duplicate keys (such as having two <string> tags with the same name attribute in the same file) will cause the Gradle resource compiler to fail during the build process.

This inspection detects these duplicates in real-time and highlights them as errors, allowing you to fix them before compiling. It also provides a quick fix to safely remove the duplicate entry.

Note: This inspection strictly checks for duplicates within the scope of the current file/module. Identical keys across different modules are perfectly valid and will not be highlighted.

Example:

<string name="app_name">My App</string>
<string name="unique_key">Welcome</string>
<string name="app_name">Duplicate App Name</string> (Error: Duplicate resource key 'app_name')

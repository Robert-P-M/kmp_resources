Reports missing or mismatched format arguments in Compose Multiplatform resource calls.

When using stringResource() or pluralStringResource(), this inspection checks the underlying XML definition for format specifiers (like %s or %1$d). If the number of arguments provided in the Kotlin code does not match the required arguments in the XML, an error is highlighted.

Note: Unlike standard Android/Java, Compose Multiplatform strictly supports only the s and d suffixes for formatting.

Example:

XML: <string name="welcome">Hello %1$s, you have %2$d messages</string>

Incorrect: stringResource(Res.string.welcome, "Rob")
(Error: expects 2 arguments, but 1 was provided)

Correct: stringResource(Res.string.welcome, "Rob", 5)
